# 第七史诗 全自动RTA（e7Rta_zd）

基于 Transformer 模型 + 计算机视觉的《第七史诗》RTA（实时竞技）全自动化系统，可自动完成**预禁用 → 选秀 → 选秀后禁用 → 战斗**的完整流程，无需人工干预。

---

## 功能概览

| 阶段 | 说明 |
|------|------|
| 大厅申请 | 自动检测大厅界面，点击申请战斗 |
| 预禁用 | 固定策略自动选择预禁英雄并确认 |
| 选秀（Draft） | Transformer 模型推荐，自动搜索并选定英雄；实时识别对手已选英雄 |
| 选秀后禁用 | 根据对手阵容，模型推荐最优禁用目标并自动执行 |
| 战斗 | 识别技能类型（AOE / 单体 / 治疗等），按优先级自动释放技能 |
| 结算确认 | 自动点击战斗结果确认，进入下一场循环 |

---

## 系统要求

- **操作系统**：Windows 10 / 11（64位）
- **模拟器**：MuMu模拟器（或雷电模拟器），分辨率设置为 **1920×1080**，窗口标题为 `第七史诗`
- **Python**：3.9 及以上

> 游戏窗口标题必须为 `第七史诗`，程序通过 Win32 API 按标题定位窗口。

## 使用方法

1. 打开模拟器，进入第七史诗，停在**大厅**或任意阶段界面。
2. 运行主程序：
   ```bash
   python auto_run_gui.py
   ```
3. GUI 启动后，点击 **▶ 开始**，程序会自动聚焦游戏窗口并开始运行。
4. 需要暂停时点击 **⏹ 停止**，当前步骤完成后自动停止；再次点击 **▶ 开始** 可从当前阶段继续。

运行日志同步写入 `run.log`，可用于调试和追踪每场选秀过程。

## 选秀识别原理

- **NCC 模板匹配**：对选秀卡和禁用槽图像与预存模板做归一化互相关，取最高分英雄作为识别结果，同时保存置信度分数（0~1）。
- **多信号修正**：
  - 搜索结果出现"已选择"字样 → 用当前英雄替换置信度最低的对手记录并置信度设为 1.0。
  - 搜索结果出现"禁用"字样 → 用当前英雄替换置信度最低的禁用记录并置信度设为 1.0。
  - 选人槽识别出的英雄出现在禁用列表中 → 判定该禁用识别有误（禁用英雄不能出现在选人槽），自动替换禁用记录并使用模型推荐的高禁用率英雄占位。
- **排除规则**：识别对手选秀槽时自动排除己方已选英雄，避免镜像误识。

## 注意事项

- 所有坐标基于 **1920×1080 客户区分辨率**，使用其他分辨率需重新标定 `executor.py` 和 `draft.py` 中的坐标常量。
- 战斗 AI 对已支持英雄（见 `skill_db/skills.json`）有最优效果；未收录英雄会退化为通用策略。
- 程序通过窗口名定位游戏，确保模拟器窗口标题包含 `第七史诗`，不要在运行期间改变窗口大小或位置。
- OCR 功能依赖 `ddddocr`，首次运行会自动下载模型，需保持网络连接。

## 常见问题

**Q：找不到窗口 `第七史诗`**  
A：检查模拟器窗口标题是否正确，或游戏是否已启动。

**Q：选秀识别率低**  
A：确认游戏画面分辨率为 1920×1080，且没有遮挡/弹窗覆盖选秀区域。

**Q：模型加载报错 `KMP_DUPLICATE_LIB_OK`**  
A：程序已在启动时自动设置该环境变量，若仍报错请在系统环境变量中手动添加 `KMP_DUPLICATE_LIB_OK=TRUE`。
